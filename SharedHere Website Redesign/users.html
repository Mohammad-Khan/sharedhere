Getting Started



To get any source code for the SharedHere project or any other project from code.google.com you need a application called “Git Bash” for Windows operating systems. It’s basically a command line interface kind of thing that interacts with Google Code's Open Source hosting. To download Git Bash, go HERE and click the green “Download GitHub for Windows” button, this should start your download. When the download finishes locate the GitHub Setup.exe file on your machine and install it. Further information on configuring your Git Bash (GitHub) can be found on the same download page under “Setup Git” section.

Once you’re done with download and the installation part of Git Bash run the following command to get a local copy of the sharedhere project git clone https://code.google.com/p/sharedhere/

Installation Instructions

To run a Google Maps-integrated application on your Android device you need to obtain the Google API add- on which includes Maps' external library. The built-in classes of the Maps Library offers you built- in downloading, rendering and caching of Maps tiles as well as a variety of display options and controls.

Installing a google API Add-on is very simple and easy process; open your SDK manager using Window -> Android SDK Manager. You can find the appropriate API under the Android version you want to use, the android version and so the API that Sharedhere uses is Android 2.3.3 version so definitely install this version. You can also download the latest version of API for your future projects. Figure 1 shows the step of how to install the appropriate API


To utilize the power of Android's Maps Library you need to obtain Google Maps Android API map key to access Google maps data. To do this you must register with the Google Maps services and agree to the listed terms and conditions. Further detailed information on how to obtain the Google Maps API key can be found at Googles' developer website with comprehensive details on the Google Maps Android API External Library HERE.


Importing the SharedHere Project into Eclipse

Start Eclipse and navigate to File -> New -> Project. You should see a folder labeledAndroid on your screen. Click the folder to expand it and select Android Project from Existing Code.


In the following window that appears, browse to the location where you cloned the project using git clone command in Git Bash and select Finish to end the process. Now you can see the project that you just created in the project explorer window on the left.

Tip: Try refactoring the project if there’s a red cross on the project and rename it; this would probably remove the error.

Since the SharedHere uses Google Maps, you have to configure changes in Eclipse to use the Google APIs to draw maps. To make the changes go to Project -> Properties and select Android along with the appropriate Google API. Tip: if the Google API is not allready installed, open the SDK manager and download the latest API.




Running SharedHere on an Android Device

Connect an Android device to your machine and install the required USB drivers if necessary. Make sure that Remote Debugging is enabled on your device so you are able to run all the apps that you build on your Android device. To enable Remote Debugging go to Settings -> Application -> Development -> Remote debugging. In Eclipse, go to Run -> run configuration -> Android Application -> Target -> Always prompt to pick a device. To run your project on a virtual device (emulator) follow instructions on this page.

Congratulations! You have successfully installed the SharedHere application to run on your Android device.
